### 작동 방식

## A5. 발표자 멘트

> *"이 그림이 오늘 발표에서 가장 중요한 한 장입니다. 위쪽이 지금의 셀사이드 리서치 흐름입니다. 매일 GS·모건스탠리·JPM·국내 증권사에서 PDF가 옵니다. 받은 사람이 한 번 읽고, 끝납니다. 며칠 뒤 같은 종목이나 같은 매크로 이슈를 다시 봐야 할 때 '그 리포트 누가 읽었더라' 부터 시작합니다.*
>
> *아래쪽이 LLM이 흡수했을 때입니다. **LLM이 본부 안에 한 명 들어온 애널리스트처럼** 매일 들어오는 리포트를 다 읽고, 본부 위키에 종목·섹터·매크로 노트로 정리합니다. 운용역이 질문하면 LLM이 위키를 보고 답하고, **그 답변이 다시 위키로 적립됩니다.** 즉 운용역이 질문하는 행위 자체가 본부 인하우스 자료를 키웁니다.*
>
> *이 패턴은 셀사이드만이 아닙니다. 종목 리서치 노트·분기 운용 보고에도 동일하게 적용됩니다. 다만 첫 파일럿은 컴플라이언스 부담이 가장 낮은 외부 발행 리서치에서 시작하는 게 안전합니다 — 그 이야기를 다음 슬라이드에서 드리겠습니다."*

> 💡 *"본부 안에 한 명 들어온 애널리스트"* 비유는 코딩 비전공 임원에게도 즉시 닿는다. 슬라이드에 박을 필요는 없고 멘트로만.

### 옵션 ㄴ — 결정 박스 중심 (1줄 + 1줄 + 박스) **[권장]**

## B6. 임원 우려 선제 차단 (자산운용 특유 항목 포함)

| 우려 | 답 |
|---|---|
| *"이거 그냥 유행 아닌가?"* | 북마크가 좋아요의 두 배(104K vs 57K). 좋아요는 '재미있다', 북마크는 '내가 적용한다'. 같은 사람이 같은 날 비전+실행 스펙 동시 공개 → 시장이 즉시 작동 시작. |
| *"4~6주에 결과가 나오나?"* | *결과* 가 아니라 *판단 가능한 지점* 에 도달. 4주차에 위키가 자라는지, 6주차에 운용역들이 실제로 쓰는지 판단. |
| *"실패하면?"* | 사라지는 건 마크다운 폴더 1개. 인프라 안 쌓고, 외부 의존 없고, 데이터는 본부 안. **롤백 자유.** |
| *"Bloomberg/FactSet 대체?"* | **대체 아니라 보조.** 외부 시스템에서 받는 PDF·메일·메모를 마크다운으로 한 겹 정리하는 단계. 두 시스템 다 사용. |
| *"MNPI/컴플라이언스?"* | 첫 파일럿은 외부 발행 리서치만 → MNPI 자체가 입력으로 들어오지 않음. 내부 자료 단계는 컴플라이언스 검토 後. on-device LLM 옵션도 가능. |
| *"규제·감사 대응?"* | 모든 답변이 raw 소스로 인용·역추적. 어떤 리포트 어느 페이지에서 나왔는지 추적. 감사 추적성 확보. |
| *"애널리스트들이 거부감?"* | 분석·판단·소통은 사람의 영역. *반복 정리* 만 LLM이 흡수. 위키가 애널리스트 작업물 색인이 되어 본인 결과물이 더 자주 인용됨. |
| *"파일럿 끝나면?"* | ① 본부 다른 영역 확장 (종목 리서치 → 분기 보고). ② 카파시가 *추가 탐색* 으로 언급한 합성 데이터·파인튜닝(본부 자료를 모델 가중치 자체에 내재화). 둘 다 6주차 보고 後 검토. |
