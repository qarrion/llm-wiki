## RAG의 한계 — 3가지 차원

### 1. 반복 (매번 처음부터)
같은 자료를 매번 처음부터 재검색한다. GS·MS·JPM 리포트 5건을 어제 종합해서 받은 답을, 오늘 같은 질문을 하면 5건을 또 처음부터 읽는다. 이전에 했던 작업이 남지 않는다.

### 2. 진화 없음 (시간이 지나도 그대로)
A기업을 2년간 분기마다 분석했다고 가정. RAG는 8개 분기 노트를 매번 처음부터 읽는다. 견해가 어떻게 바뀌어 왔는지, 왜 바뀌었는지는 관심 없다. 시간이 지나도 지식은 깊어지지 않는다.

### 3. 관계 무지 (문서 간 연결을 모름)
매크로팀이 "중국 경기 둔화" 리포트를 올렸다. 리서치팀이 "LVMH 실적 견조" 노트를 따로 올렸다. RAG는 두 파일을 별개로 취급한다. LVMH가 중국 소비에 노출되어 있다는 것, 그게 현재 포트폴리오의 어느 포지션과 연결되는지 모른다.

## 확정 화면 구성

**① 헤드라인 (큰 글씨)**
> 지금 방식의 LLM에는 세 가지 한계가 있다.

**② 보조 한 줄 (작은 글씨)**
> RAG · NotebookLM · ChatGPT 파일 업로드 — 모두 같은 구조.

**③ 중앙 다이어그램 — 3가지 한계를 흐름으로**

```
① 반복
   질문 (월)  →  GS·MS·JPM 리포트 5건 재검색  →  답변  →  ⨯ 휘발
   질문 (화)  →  GS·MS·JPM 리포트 5건 재검색  →  답변  →  ⨯ 휘발   ← 똑같은 비용, 매번

② 진화 없음
   Q1 A기업 분석  ─┐
   Q2 A기업 분석  ─┤  →  8개 파일 더미  →  답변  →  ⨯ 흐름 모름
   ...Q8 분석     ─┘                                  ← 2년치 견해 변화, 어디에도 없음

③ 관계 무지
   매크로팀: "중국 경기 둔화"  ─┐
                                  ├→  별개 파일 2개  →  답변  →  ⨯ 연결 모름
   리서치팀: "LVMH 실적 견조"  ─┘                        ← 포트폴리오 익스포저, 누가 연결하나
```

**④ 하단 우측 — Karpathy 인용 (작은 글씨)**
> "이전에 했던 작업이 남지 않기 때문에, 매번 동일한 과정을 반복하게 된다." — Karpathy

**⑤ 슬라이드 3 연결 한 줄 (좌하단)**
> → 단순 요약에서 멈춘다.

## 발표자 멘트

> *"지난 한 주만 떠올려 보셔도, LLM에 같은 자료를 몇 번이나 다시 올렸는지 기억하실 겁니다. 매번 LLM은 새로운 인턴처럼 행동합니다. GS·모건스탠리 리포트 5건을 종합해서 어제 받은 답을, 오늘 같은 질문을 하면 또 5건을 처음부터 읽습니다. 그동안 정리해둔 분석은 채팅 로그 어딘가에 있고, 다음에 같은 답이 다시 나온다는 보장도 없습니다.*
>
> *사실 반복보다 더 근본적인 문제가 둘 있습니다. 시간이 지나도 지식이 깊어지지 않는다는 것, 그리고 문서 사이의 관계를 모른다는 것입니다. 다음 슬라이드에서 LLM Wiki가 이 세 가지를 어떻게 바꾸는지 보여드리겠습니다."*
