# 최종 v1 — 슬라이드 1 (page01.png 역추출)

## 이 페이지의 정체

### 목적 — 무엇을 위한 한 장인가

발표 전체의 **훅(hook) + 개념 정의 + 권위 예고** 를 한 화면에서 동시에 처리하는 슬라이드.
임원이 *"LLM-WIKI? 처음 듣는 말인데"* 부터 *"아 — Claude랑 Obsidian 합치면 저런 그림이 나오는구나, Karpathy가 띄웠고"* 까지 5초 안에 흡수하도록 설계.

발표자 입장에서는 *"오늘 자리는 본부가 한 번 들여다본 결과를 공유하는 자리"* 라는 입장 표명 기회.
청중 입장에서는 *"이게 뭐고, 어떻게 만들고, 결과물은 어떻게 생겼고, 누가 만들었나"* 의 4가지 의문을 한 장으로 처리받음 — 슬라이드 2 이후로 *"기본 개념 다시 설명해야 하나"* 부담 없이 진입 가능.

### 설명 대상 — 무엇을 설명하는가

이 한 장이 임원의 머리에 박아 넣는 **4가지 정보** :

| #   | 정보                                                              | 화면 위치         |
| --- | --------------------------------------------------------------- | ------------- |
| ①   | **개념 이름** — *LLM-WIKI* (방식 자체에 붙은 이름)                           | 상단 제목, 컬러 강조  |
| ②   | **WIKI 어원 정의** — *공동 관리 지식 시스템* (위키 단어의 본뜻 환기)                  | 부제, 회색 한 줄    |
| ③   | **조립 공식** — *Claude(LLM) + Obsidian(도구) = Knowledge Graph(결과물)* | 중앙 수식형 3단 비주얼 |
| ④   | **권위 예고** — *Karpathy 진원지* (다음 장 본격 권위로 다리 놓기)                  | 좌하단, 작게       |

### 설명 방식 — 어떻게 설명하는가

1. **이름 → 정의 → 공식 → 결과물** 의 위→아래 정보 위계.
   - 위에서 *이름* 을 박고, 바로 아래 *정의* 로 위키라는 단어의 모호함을 제거하고, 중앙에서 *공식* 으로 만드는 방법을 보여주고, 우측의 *결과물 그림* 으로 *"이런 게 나옵니다"* 를 시각화.
2. **이중 수식 운율로 시각 일관성 확보.**
   - 중앙: `Claude + Obsidian = Knowledge Graph` (도구 조립)
   - 우하단: `점(노트) + 선(LLM이 발견한 연결) = 인사이트, 직관` (데이터 조립)
   - 같은 `A + B = C` 패턴이 두 번 반복되면서 *"이 슬라이드 = 조립의 그림"* 으로 임원 머릿속에 박힘.
3. **부제에서 *"WIKI" 어원* 을 정의하는 이유.**
   - 임원 중 *"위키가 정확히 뭐였더라"* 하는 사람을 잡아주는 한 줄.
   - *"공동 관리"* 라는 부분이 발표 전체의 *왜 지금* 으로 자연스럽게 이어짐 — *"그동안 위키가 안 됐던 이유는 사람이 공동 관리 비용을 못 감당해서. 그 자리를 LLM이 받는다."* (발표자 멘트에서 회수).
4. **권위 시그널을 좌하단에 *작게* 두는 이유.**
   - 1장에서 권위로 *짓누르지 않기* — *"Karpathy 한 분 이름이 다음 장에서 자세히"* 정도의 예고만. 본격 권위·시장 검증·적기성은 슬라이드 2에서 풀이.
5. **수식형 비주얼이 단일 그래프 그림보다 강한 이유.**
   - 단일 그래프 그림은 *"멋있다"* 까지만 감.
   - *Claude + Obsidian = Knowledge Graph* 수식은 *"이걸 만들려면 무엇이 필요한가"* 를 즉시 답함 → 슬라이드 6(도구 얘기)으로 가는 다리를 미리 깔아둠.

### 발표 후 청중 머릿속에 남길 한 줄

> *"LLM-WIKI라는 게 있네. Claude랑 Obsidian 합치면 저런 그래프가 나오고. Karpathy가 띄웠고. 본부가 한 번 들여다보는 자리구나."*

## 슬라이드 1 — LLM-WIKI : LLM이 직접 자신만의 위키를 만든다.

### 화면 구성 (위 → 아래)

**① 상단 — 제목 (큰 글씨, 한 줄)**

> **LLM-WIKI** : LLM이 직접 자신만의 위키를 만든다.

(*"LLM-WIKI"* 부분만 컬러 강조 — 오렌지 톤)

**② 부제 (작게, 한 줄, 회색)**

> **WIKI**: 다수의 사용자가 자유로이 변경된 문서를 작성하고 공동 관리하는 지식 시스템

(WIKI 단어 어원·정의 회수 — *"위키"* 라는 단어 자체를 임원에게 다시 정의해주는 한 줄)

**③ 중앙 — 수식형 3단 비주얼 (가로 배치)**

```
┌─────────────┐         ┌─────────────┐         ┌──────────────────────┐
│             │         │             │         │   Knowledge Graph    │
│   [Claude   │    +    │  [Obsidian  │    =    │                      │
│    아이콘]   │         │   아이콘]    │         │   (옵시디언 graph    │
│             │         │             │         │    view, 다크 배경,  │
│   Claude    │         │  Obsidian   │         │    컬러 노드 클러스터) │
│ AI 어시스턴트 │         │ 제작·관리 도구│         │                      │
└─────────────┘         └─────────────┘         └──────────────────────┘
   (다크 카드)            (다크 카드)                (큰 그래프 이미지)
```

- **Claude 카드 (좌):** 다크 배경 카드. 오렌지 Claude 로고 + *"Claude / AI 어시스턴트"* 라벨.
- **`+` 기호:** 카드 사이 가운데, 큰 글씨.
- **Obsidian 카드 (가운데):** 다크 배경 카드. 보라색 Obsidian 로고 + *"Obsidian / 제작·관리 도구"* 라벨.
- **`=` 기호:** Obsidian 카드와 Knowledge Graph 사이.
- **Knowledge Graph (우):** 옵시디언 graph view 이미지 (다크 배경, 컬러 그룹 노드, dense 코어 + 외곽 가지). 상단에 *"Knowledge Graph"* 라벨.

**④ 좌하단 — 권위 시그널 (작게, 회색)**

> 진원지: Andrej Karpathy (전 Tesla AI 디렉터·OpenAI 창립멤버)
> 2026-04-03 · X 20.6M Views · Gist ⭐5,000+
> *— 다음 장에서 자세히.*

**⑤ 우하단 — 수식형 캡션 (작게)**

> 점(노트) + 선(LLM이 발견한 연결) = 인사이트, 직관

(③ 중앙 비주얼의 *"Claude + Obsidian = Knowledge Graph"* 수식과 운율 일치 — 이중 수식 회수)

**⑥ 좌측 푸터 — 페이지 번호**

### 이 한 장이 전달하는 것

1. **이름·정의·결과물 한 화면 동시 제시.**
   - 이름: **LLM-WIKI** (제목)
   - 정의: **WIKI = 공동 관리 지식 시스템** (부제)
   - 결과물: **Knowledge Graph** (우측 비주얼)
2. **수식형 비주얼로 *조립 관계* 명확화.** *Claude(LLM) + Obsidian(도구) = Knowledge Graph(결과물)* — 임원이 5초 안에 *"이 셋이 어떻게 연결되는가"* 인지.
3. **이중 수식 운율.** 중앙 비주얼 *"Claude + Obsidian = Knowledge Graph"* 와 우하단 캡션 *"점 + 선 = 인사이트, 직관"* 이 같은 `A + B = C` 패턴. 본부 일과 연결되는 메타포: *도구를 합치면 → 결과물이 나온다*.
4. **권위 예고 (좌하단).** *"이게 그냥 유행이 아니다"* 신호. 본격 권위는 다음 장에서.

### 발표자 멘트

> *"최근 'LLM이 노트를 직접 정리한다 — 그래서 직접 위키를 만든다' 는 프레임워크가 개발자 사이에서 화제입니다. 저도 딱 개념을 듣자마자 *'이건 진짜 물건이다'* 싶었는데요, 이번 기회에 잘 정리해서 공유하는 자리를 가지게 되었습니다.*
>
> ***클로드 코드와 옵시디언, 둘 다 새로운 개념은 아닙니다.*** *조금 설명드리자면 — 클로드 코드는 다들 아시는 *CLI에서 LLM을 다루는 대표 서비스* 입니다. 옵시디언은 LLM보다 훨씬 오래된 서비스인데, 특이한 점은 공교롭게도 *LLM이 자주 사용하는 텍스트 파일 형식인 마크다운* 으로 노트를 만드는 서비스라는 점입니다.*
>
> *옵시디언의 주요 기능은 — 문서 사이의 *링크* *백링크* 라는 기능을 통해 문서와 문서를 양방향으로 연결할 수 있어서, **21세기 지식 노동자들의 *데이터 상실에 대한 근원적인 불안감* 을 해소해주는 기능** 으로 널리 사용되어 왔습니다.*
>
> *다만 — 한 번이라도 노트류 서비스를 정리해 보셨으면 아시겠지만, **정리할 문서가 많아질수록 사용자가 감당해야 하는 *북키핑 비용* 이 너무 크다는 점인데요** 
> 
> (*그 북키핑 비용을 LLM이 받아낸다 — 가 어떻게 보면 하나의 포인트인데. 이 이야기는  진원지부터 짚고 들어가겠습니다."*)

마지막 문장이 슬라이드 2(Karpathy 진원지) 로의 다리.

**이전 페이지에서 이번 페이지로의 흐름:**
- *"노트가 정리된다 → 위키가 만들어진다"* 프레임워크 소개 (화제성)
- 발표자 개인 반응 *"이건 진짜 물건이다"* (자리 정당성 — 왜 오늘 이 자리)
- **클로드 코드 / 옵시디언 = 둘 다 기존 도구** (중앙 비주얼 두 카드 풀이)
- 옵시디언 깊이 풀이 — 마크다운 + 링크/백링크 → *21세기 지식 노동자 데이터 상실 불안감 해소* (왜 옵시디언이 사랑받았나)
- **단점 = 북키핑 비용** (전통 위키의 한계 — 발표 전체의 *통증* 신호)
- → *"그 비용을 LLM이 받아낸다"* — 슬라이드 2 진원지(Karpathy) 로의 다리

### 슬라이드 2와의 연결

슬라이드 1의 좌하단 권위 한 줄(*"진원지: Andrej Karpathy ... — 다음 장에서 자세히"*) + 발표자 멘트 마지막 문장(*"진원지부터 짚고 들어가겠습니다"*) 이 **두 갈래 다리**.

→ 슬라이드 2 헤드라인이 그대로 이어짐. *"andrej karpathy (전 Tesla AI 디렉터, OpenAI 창립멤버) — '코드 조작보다 더 많은 토큰을 쓰고 있다'"*. 자연스럽게 *권위 본격 소개 + 적기성·시장 검증* 으로 흘러감.

청중 머리 속 흐름:
1. *"LLM-WIKI? 처음 듣는 말인데 그림이 멋있네"* (제목 + Knowledge Graph)
2. *"아 — Claude + Obsidian 조립이구나. 위키 정의도 부제에 있네"* (수식 + 부제)
3. *"근데 누가 만든 거?"* (자동으로 발생하는 의문)
4. *"아, Karpathy라는 사람의 4월 글이 진원지구나"* (슬라이드 2에서 답)
5. *"그래서 우리 일과 무슨 상관?"* (슬라이드 3 통증으로 자동 진입)
