## 이 페이지의 정체

### 목적 — 무엇을 위한 한 장인가

슬라이드 1에서 던진 *"LLM-WIKI"* 라는 명칭에 **권위·적기성·시장 검증** 을 한 화면에서 입혀주는 슬라이드.
임원이 *"누가 이걸 띄웠고, 언제, 얼마나 진지하게 받아들여지고 있는가"* 의 3가지 의문을 한 장으로 해소받음.

발표자 입장에서는 *"이게 그냥 유행이 아니다"* 를 **숫자와 1차 원본** 으로 증명할 수 있는 자리.
청중 입장에서는 *"이 명칭이 어디서 왔는지"* (= GitHub Gist `llm-wiki.md` 파일명) 가 동시에 풀이됨 → 슬라이드 1의 명칭이 *작명* 이 아니라 *인용* 이었음을 확인.

### 설명 대상 — 무엇을 설명하는가

이 한 장이 임원의 머리에 박아 넣는 **4가지 정보** :

| # | 정보 | 화면 위치 |
|---|---|---|
| ① | **인물** — Andrej Karpathy (Tesla AI 디렉터·OpenAI 창립멤버) | 상단 헤드라인 |
| ② | **시점** — 2026-04-03, 한 트윗 | 상단 헤드라인 (날짜 명시) |
| ③ | **비전 + 실행 동시 공개** — X 트윗(좌) + GitHub Gist `llm-wiki.md`(우) 같은 날 | 중앙 좌·우 스크린샷 2장 |
| ④ | **시장 검증** — X 20.6M Views / Gist ⭐5,000+ / 북마크 104K | 스크린샷 하단 숫자 |

### 설명 방식 — 어떻게 설명하는가

1. **헤드라인 = 인물 + 시점 한 줄.**
   - *"진원지는 Andrej Karpathy. 4월 3일, 한 트윗에서 시작."* — 누구·언제·어디서 가 한 줄에 다 들어감. 임원이 5초 안에 *"3주 전 한 사람의 트윗"* 으로 인지.
2. **헤드라인 옆 인용 = Karpathy 본인 발언.**
   - *"코드 짜는 시간보다 위키를 키우는 데 더 많은 시간을 쓰고 있다. 꽤 잘 작동한다."* — *바이브코딩* 의 대명사가 *코드 대신 위키* 라고 한 점이 가장 강력한 시그널. 임원에게 *"이 사람이 이걸 진심으로 한다"* 박는 한 줄.
3. **좌·우 스크린샷 2장으로 *비전 + 실행* 동시 공개 입증.**
   - 좌: X 트윗 (비전 — *"using LLMs to build personal knowledge bases"*)
   - 우: GitHub Gist `karpathy / llm-wiki.md` (실행 스펙 — 같은 날 공개된 실행 지침)
   - **같은 날** 비전과 실행 스펙이 동시에 나왔다는 건 보통 일이 아님 — *"바로 적용 가능"* 신호.
4. **`llm-wiki.md` 파일명 = 슬라이드 1 제목의 출처.**
   - 우측 Gist 스크린샷의 파일명이 시각적으로 강조됨 → 임원이 *"아, 슬라이드 1의 LLM-WIKI 명칭이 작명이 아니라 인용이구나"* 즉시 인지.
5. **숫자가 *유행* 이 아닌 *적용 의지* 임을 증명.**
   - X 조회 20.6M, Star 5,000+, **북마크 104K (좋아요 57K의 거의 두 배)**.
   - *좋아요 = "재미있다"* / *북마크 = "내가 직접 적용해보겠다"* — 시장이 진지하게 움직이고 있다는 한 가지 신호로 *북마크 숫자* 를 강조.

### 발표 후 청중 머릿속에 남길 한 줄

> *"3주 전 Karpathy 트윗 하나에서 시작됐고, 비전 + 실행 스펙이 같은 날 공개됐고, 시장이 이미 진지하게 움직이고 있다. *유행* 이 아니라 *적용* 단계."*
