# 최종 v1 — 슬라이드 3 (page03.png 역추출)

## 이 페이지의 정체

### 목적 — 무엇을 위한 한 장인가

발표 전체의 **통증(Pain) 슬라이드** — 슬라이드 1·2 에서 *명칭과 진원지* 를 박았다면, 여기서 *"왜 이게 필요한가"* 를 임원에게 직격하는 자리.
임원이 이미 사용하고 있는 **RAG · NotebookLM · ChatGPT** 를 직접 비교군으로 호명하면서 *"우리 이미 쓰는 도구의 한계"* 를 인정하게 만듦.

발표자 입장에서는 *"같은 LLM이라도 *구조* 가 다르면 답이 다르다"* 는 핵심 메시지를 **비교표 + 살아있는 답변 한 쌍** 으로 한 번에 전달.
청중 입장에서는 *"우리 이미 ChatGPT 쓰는데 뭐가 다른가"* 라는 자연스러운 의문이 비교표·답변 예시로 *동시에 풀이* 됨 → 슬라이드 4 (구조 풀이)로 부담 없이 진입.

### 설명 대상 — 무엇을 설명하는가

이 한 장이 임원의 머리에 박아 넣는 **4가지 정보** :

| # | 정보 | 화면 위치 |
|---|---|---|
| ① | **통증** — *지식이 축적되지 않는다* (현재 RAG/NotebookLM/ChatGPT 공통 한계) | 헤드라인 직후 검정 띠 |
| ② | **4가지 차이** — raw 재검색 / 답변 휘발 / 매번 처음부터 / 시간 지나도 그대로 | 중앙 비교표 (좌 단순 요약 → 우 LLM-WIKI) |
| ③ | **살아있는 답변 한 쌍** — 같은 질문, 다른 답 (좌 1줄 / 우 4줄) | 표 아래 Q&A 블록 |
| ④ | **AI slop 낙인** — 단순 요약은 *"ai slop"* 이라는 명시적 비판 | 좌측 답변 끝 빨간 글씨 |

### 설명 방식 — 어떻게 설명하는가

1. **헤드라인 = 질문형 도발.**
   - *"자료는 쌓이고 있다. 이해는 쌓이고 있는가?"* — *이해* 부분만 오렌지 강조. 임원이 본부 일상에서 느끼는 *"자료는 많은데 정작 손에 잡히는 게 없다"* 감각을 정조준.
2. **헤드라인 직후 검정 띠 = 통증 한 줄 + 도구 호명.**
   - *"매번 업로드한 자료에서 지식을 다시 발견할 뿐, 축적되지 않는다."* + *"현재의 RAG · NotebookLM · ChatGPT 모두 같은 한계."*
   - **세 도구를 직접 호명** 하는 게 핵심 — 임원이 *"막연한 비판"* 으로 받지 않고 *"내가 어제 쓴 그 도구"* 로 받게 함.
3. **비교표 = 4행, 좌·우 헤더 색 대비.**
   - 좌측 헤더 *"단순 요약 (RAG / NotebookLM / ChatGPT)"* 회색 띠 — *기존, 익숙함, 빛바램*.
   - 우측 헤더 *"LLM-WIKI"* 검정 띠 — *새로움, 단정, 무게*.
   - 행마다 좌→우 화살표(→) 가 *"변환됨"* 을 시각화. 4행 모두 같은 패턴 → 비교 흡수가 1.5초 안에 끝남.
4. **표 아래 살아있는 답변 = 부피 비대칭.**
   - 좌측 *"한 보고서 인용으로 끝"* + **`ai slop`** 빨간 낙인 — 단 두 줄.
   - 우측 *컨센서스 종합 + 모순 표시 + 태그* — 네 줄.
   - **부피 차이 자체가 비교표 4행("시간이 지날수록 풍부해짐")의 시각적 회수.** 좌·우 표 형태로 두면 셀이 균등해서 이 차이가 안 보임 → 수직 부피 비교가 정답.
5. ***"ai slop"* 빨간 낙인 = 새로운 무기.**
   - 단순 요약이 그냥 *"부족한 답"* 이 아니라 ***"AI slop"*** (생성 AI 의 의미 없는 양산물) 이라고 명시.
   - 임원에게 *"이미 쓰는 도구가 만드는 답의 정체"* 를 한 단어로 박아주는 효과 — 슬라이드 4·5 *제대로 된 답을 만드는 구조* 의 대비를 강하게.
6. **Karpathy 인용 / *"단순 요약에서 멈춘다"* 다리 한 줄 — 둘 다 삭제.**
   - 초안 v5 의 두 요소가 빠지고, 한 화면에 *통증 + 비교표 + 답변 한 쌍* 의 3블록 구성으로 압축.
   - 인용 권위는 슬라이드 2에서 이미 무겁게 박혔으므로 여기서는 *기능 비교* 에만 집중.

### 발표 후 청중 머릿속에 남길 한 줄

> *"내가 어제 쓴 ChatGPT/NotebookLM 답변 — 그게 ai slop 이었구나. LLM-WIKI는 같은 질문에 4줄짜리 답을 *모순 표시까지 포함해서* 준다."*

## 슬라이드 3 — 자료는 쌓이고 있다. 이해는 쌓이고 있는가?

### 화면 구성 (위 → 아래)

**① 상단 — 헤드라인 (큰 글씨, 한 줄)**

> **"자료는 쌓이고 있다. 이해는 쌓이고 있는가?**

(*"이해는 쌓이고 있는가?"* 부분만 오렌지 강조)

**② 헤드라인 직후 — 통증 띠 (검정 배경, 흰 글씨, 한 줄)**

> 매번 업로드한 자료에서 지식을 다시 발견할 뿐, **축적되지 않는다.** 현재의 RAG · NotebookLM · ChatGPT 모두 같은 한계.

(*"축적되지 않는다"* 부분만 오렌지 강조)

**③ 중앙 — 비교표 (좌·우 4행, 헤더 색 대비)**

```
┌─────────────────────────────────────┐    ┌─────────────────────────────────────┐
│ ▌ 단순 요약                         │    │ ▌ LLM-WIKI                          │
│   (RAG / NotebookLM / ChatGPT)      │    │                                     │
│   (회색 헤더 띠)                    │    │   (검정 헤더 띠)                    │
├─────────────────────────────────────┤    ├─────────────────────────────────────┤
│ 매 질문마다 raw 재검색·재조립        │ →  │ 한 번 컴파일된 지식이 누적          │
├─────────────────────────────────────┤    ├─────────────────────────────────────┤
│ 답변은 채팅 로그로 휘발              │ →  │ 좋은 답변은 위키 페이지로 적립      │
├─────────────────────────────────────┤    ├─────────────────────────────────────┤
│ 5문서 종합 → 매번 처음부터          │ →  │ 교차 참조·모순 표시 이미 완료       │
├─────────────────────────────────────┤    ├─────────────────────────────────────┤
│ 시간이 지나도 그대로                │ →  │ 시간이 지날수록 풍부해짐            │
└─────────────────────────────────────┘    └─────────────────────────────────────┘
```

- **좌측 (단순 요약):** 회색 헤더 띠 + *(RAG / NotebookLM / ChatGPT)* 부제. 4행 본문, 익숙한 한계 일람.
- **화살표 (→):** 행마다 좌→우 오렌지 화살표. *"이렇게 바뀐다"* 의 시각화.
- **우측 (LLM-WIKI):** 검정 헤더 띠. 강조어 (누적 / 적립 / 완료 / 풍부해짐) 오렌지.

**④ 표 아래 — 살아있는 답변 한 쌍 (수직 비대칭, 좌 짧음·우 김)**

```
Q. "삼성전자 내년 목표가는?"

▶ 단순 요약 답변 (RAG / NotebookLM / ChatGPT)
   "GS 리포트(4/22)에 따르면 목표가 95,000원으로 상향."
   → 한 보고서 인용으로 끝, 다음 질문에 또 처음부터.       [ai slop]   ← 빨간 낙인

▶ LLM-WIKI 의 답변
   · 컨센서스 95K (GS 95 / MS 92 / JPM 98) — 3개월 전 평균 88K 대비 +7K 상향
   · 상향 근거: HBM 신규 캐파 가시화                      [#메모리/HBM]
   · 본부내부는 4월 약세(컨센서스와 모순) ⚠ 환율가정차이  [#매크로/USDKRW]
     → 위키 내부의 문서의 정리 / 관계에 따라 답이 생성됨.
```

- **좌측 답변:** 의도적으로 1~2줄. 끝에 빨간 글씨 **`ai slop`** 낙인.
- **우측 답변:** 4줄. 컨센서스 / 근거 / 본부 모순 / 마무리. 태그 (#메모리/HBM, #매크로/USDKRW) 가 *위키 내부 연결* 을 시각화.
- **부피 비대칭** = 비교표 4행의 시각적 회수.

**⑤ 좌측 푸터 — 페이지 번호**

### 발표자 멘트

> *"먼저 글을 읽어보면, 이 글은 기존 LLM 사용에 대한 문제에 대해 이야기하고 있습니다. **결국에는, 질문을 하면 또 처음부터 시작합니다.** 매번 업로드한 자료에서 지식을 *다시 발견* 할 뿐, **축적되지 않는다** 는 것입니다. 이게 소위 말하는 *ai slop* 의 원인이기도 합니다.*
>
> *비교표를 통해 좀 더 자세하게 설명드릴 수 있습니다. 왼쪽이 지금 쓰는 방식 — 매 질문마다 재검색·재조립, 답변은 휘발, 5문서 종합도 매번 처음부터, **시간이 지나도 지식은 그대로** 입니다. 그러나 오른쪽 — **LLM-WIKI** 를 사용하면 *개념적으로 더 나은 대답* 이 가능하다는 주장입니다.*
>
> *예를 들어 설명해보겠습니다. 아래 *'삼성전자 내년 목표가는?'* 이라는 같은 질문에 — 단순 요약은 *'GS 리포트에 따르면 95K로 상향'* 한 줄로 끝납니다. **전형적인 *ai slop* 입니다.** 생성형 AI가 만든 그럴듯하지만 깊이 없는 답. *존댓말로 욕 들은 기분.* LLM-WIKI는 같은 질문에 — 컨센서스 종합부터 본부 내부 의견과 컨센서스의 *모순* 까지 자동으로 정리해서 답합니다. 모순의 원인 (환율 가정 차이) 까지 짚고, 관련 위키 페이지 (#메모리/HBM, #매크로/USDKRW) 로 자동 연결됩니다. **중요한 건 — 의도된 방향으로 데이터를 축적해서 지식을 만들었다는 점입니다.***
>
> *같은 LLM 입니다. **구조가 다를 뿐.** 다음 장에서 좀 더 자세하게 알아보겠습니다."*

### 임원 Q&A 대비

| 예상 질문 | 답 |
|---|---|
| *"NotebookLM이랑 뭐가 다른가? 우리 이미 쓰고 있는데"* | "NotebookLM은 *raw 문서만 보고 매번 처음부터* 답합니다. 답변은 채팅 안에 남고요. 비교표 ②번 — 답변이 위키로 적립되지 않으면 *다음 질문도 처음부터* 입니다. LLM-WIKI는 그 답변이 다시 위키로 들어갑니다." |
| *"ChatGPT / Claude 히스토리 기능 있지 않나?"* | "대화 기록은 남습니다. 그게 *다음 질문의 출발점* 이 되지는 않습니다. 매번 새 대화입니다. *기록* 과 *컴파일된 지식* 은 다른 것입니다." |
| *"RAG 시스템 도입 고려 중인데"* | "RAG는 비교표 *왼쪽* 입니다 — 질문마다 raw에서 chunk 가져옴. 정리·축적은 안 됩니다. RAG와 LLM-WIKI는 *대체* 가 아니라 *층위 다름* — RAG가 raw 검색이라면 Wiki는 *정리된 지식 단위*. 같이 쓸 수도 있습니다." |
| *"5문서 종합? 우리 자료는 더 많은데"* | "숫자는 예시입니다. 50문서든 500문서든 본질은 같음 — *매번 처음부터 종합* 하는 vs 이미 *교차 참조·모순 표시가 되어 있는* 위키에서 답하는 차이입니다." |
| *"비교표 왼쪽 도구들도 결국 LLM 아닌가? 같은 LLM이 왜 다르게 동작?"* | "도구가 다른 게 아니라 *구조* 가 다릅니다. 같은 LLM이 *raw에서 매번 답을 만들면* RAG, *답을 위키로 다시 쓰게 시키면* LLM-WIKI 입니다. 같은 엔진, 다른 사용 패턴." |
| *"답변 예시의 LLM-WIKI가 결국 종목 추천 아닌가?"* | "**아닙니다.** 위키는 *의견 종합·변화·모순 표시* 까지 — 결론(*'사야 한다'*)은 만들지 않습니다. 답변에 *모순* 까지만 표시되고, *판단은 사람* 입니다. 슬라이드 5·6에서 그 선이 더 명확해집니다." |
| *"`ai slop` 표현이 너무 거친 것 아닌가?"* | "정확한 단어를 골랐습니다. 이미 업계에서 *그럴듯하지만 깊이 없는 생성 AI 답* 을 *AI slop* 이라고 부릅니다. *우리가 일상적으로 받는 답이 그 카테고리* 라는 점을 임원분들께 분명히 인지시키고 싶었습니다. 부드럽게 말하면 메시지가 흐려집니다." |

### 슬라이드 4와의 연결

**다리 한 줄 (발표자 멘트 마지막):**

> *"같은 LLM, 다른 구조. 그 구조가 정확히 어떻게 생겼는지 — 다음 장에서 보여드립니다."*

→ 슬라이드 4 헤드라인으로 자동 진입. *비교표 우측 LLM-WIKI 4가지 차이* 가 *"어떤 구조에서 만들어지는가"* 로 풀이됨.

청중 머리 속 흐름:
1. *(슬라이드 2)* "Karpathy가 띄웠고, 시장이 이미 적용 단계."
2. *(슬라이드 3 헤드라인)* "이해는 쌓이고 있는가? — 본부 일상의 그 답답함이 맞다."
3. *(검정 띠)* "ChatGPT, NotebookLM — 내가 어제 쓴 그 도구네."
4. *(비교표)* "왼쪽 4가지가 정확히 내 일상. 오른쪽이 LLM-WIKI."
5. *(답변 예시)* "삼전 목표가 — 같은 질문에 답이 이렇게 다르다. **ai slop 이라는 말이 강하지만 부정 못 하겠다.**"
6. *(자동 의문)* "같은 LLM인데 왜 답이 이렇게 다른지 — 그 구조가 뭔지 보여달라."
